# Contributing

Thanks for helping build Claw95.

## Setup
1. Install Python 3.11+
2. Create venv
3. Install deps
4. Run tests and linters

## Branch Strategy
- `main` is stable
- feature branches: `feat/<short-name>`
- fix branches: `fix/<short-name>`

## Commit Style
Use Conventional Commits:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code structure change
- `test:` tests

## Pull Requests
A PR should include:
- problem statement
- design decision
- test coverage
- docs updates (if behavior changed)

## Definition of Done
- tests pass
- lint passes
- docs updated
- audit impact reviewed
